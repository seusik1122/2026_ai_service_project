## 3. 합치기 작업 순서 (B 기준)

> **목표**: `/mnt/c/ai_service/` 를 최종 통합 저장소로 사용

### Step 1 — `.env` 동기화 (5분)
```bash
# B .env에 A 전용 키 추가
echo "OPENAI_DAILY_LIMIT=500" >> /mnt/c/ai_service/backend/.env
echo "PORT=8000" >> /mnt/c/ai_service/backend/.env
```

### Step 2 — A의 독점 파일 복사 (2분)
```bash
cp /mnt/c/ai_service/lecture-recommendation/backend/scripts/seed_demo_data.py \
   /mnt/c/ai_service/backend/scripts/  # scripts/ 폴더 없으면 mkdir
cp /mnt/c/ai_service/lecture-recommendation/backend/tests/test_api_integration.py \  # 이미 있음
   # → B에 이미 존재, 스킵
```

### Step 3 — requirements.txt 확인 (1분)
- B 기준 그대로 사용 (openai==2.40.0 유지)
- A 쪽 `openai==1.40.0`은 무시

### Step 4 — 프론트엔드 `.env` 수정 (2분)
```bash
# /mnt/c/ai_service/lecture-recommendation/frontend/.env 중복 제거
# VITE_INTERNAL_API_KEY 값을 B의 API_SECRET_KEY(my-secret-1234)로 교체
```

### Step 5 — 통합 동작 확인
```bash
# 터미널 1: FastAPI 서버
cd /mnt/c/ai_service/backend && uvicorn main:app --reload --port 8000

# 터미널 2: 통합 테스트
cd /mnt/c/ai_service/backend && python -m pytest tests/test_api_integration.py -v

# 터미널 3: 프론트엔드
cd /mnt/c/ai_service/lecture-recommendation/frontend && npm run dev
```

## 6. Claude Code 효율화 제안

### 현재 문제
- B(`/ai_service/`)와 A(`/lecture-recommendation/`)가 **두 개의 분리된 저장소**처럼 운영됨
- CLAUDE.md가 `/ai_service/lecture-recommendation/`에만 있어 `/ai_service/backend/` 작업 시 규칙 적용 안 됨

### 제안

**① `/ai_service/CLAUDE.md` 생성 (B 작업 영역용)**
```markdown
# 강의 추천 시스템 — B담당 작업 규칙
- 작업 전 /ai_service/tasks.md 확인
- 백엔드 루트: /mnt/c/ai_service/backend/
- 프론트엔드: /mnt/c/ai_service/lecture-recommendation/frontend/
- DoD: pytest tests/ -v 통과 + /docs 200 응답
```

**② `tasks.md` 단일화**
- 현재 두 파일이 **서로 다른 완료 상태**를 표시 중 (혼동 원인)
- `/ai_service/tasks.md`를 마스터로 사용, lecture-recommendation/tasks.md는 삭제

**③ `test_api_integration.py` 를 4주차 연동 검증의 기준으로**
- 현재 B에만 있으므로 합치기 후 이 파일 하나로 "연동 완료" 판정 가능
